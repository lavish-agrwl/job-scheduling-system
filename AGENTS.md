## Coding Style & Naming Conventions

- **Indentation:** 2 spaces, no tabs.
- **Files:** Use kebab‑case (`job-scheduler.ts`).
- **Classes/Interfaces:** PascalCase (`JobScheduler`).
- **Variables/Functions:** camelCase (`scheduleJob`).
- **Constants:** UPPER_SNAKE_CASE (`MAX_RETRIES`).
- Run `bun fmt` or `npm run lint -- --fix` before committing.

## Testing Guidelines

- **Framework:** Jest (configured via `package.json`).
- **File naming:** `*.test.ts` placed alongside the module or under `tests/`.
- **Coverage:** Aim for ≥80 % line coverage; CI will fail below this threshold.
- Run `bun test --coverage` locally to view the report.

## Commit & Pull Request Guidelines

- **Commit messages:** Follow Conventional Commits (`type(scope): description`). Example: `feat(scheduler): add priority queue`.
- **PR description:** Summarize changes, reference related issue (`Fixes #42`), and list any breaking changes.
- **PR checks:** Must pass lint, build, and test steps. Include screenshots only for UI‑related changes.
